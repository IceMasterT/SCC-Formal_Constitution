# CI evidence lock

A final release should include a CI artifact named `scc-kernel-release-gate-evidence` generated by `.github/workflows/ci.yml`. The artifact should contain:

- `release_gate_transcript.txt`: full output from `scripts/run_release_gates.sh`;
- `release_manifest.json`: SHA-256 release manifest generated after validation.

A release should not claim a Rust pass unless the transcript contains:

```text
cargo test --all-features ... ok
cargo test --test negative_corpus ... ok
cargo test --test golden_vectors ... ok
cargo test --test proptest_invariants ... ok
release_gate=PASS
```

This file is included to prevent evidence drift: release claims must refer to a concrete transcript rather than an implied or unobserved command.
